# Discord Bot & Admin Dashboard

This repository contains a full-stack production-ready web application for a Discord slash command bot, complete with an Interactions Endpoint and a React-based Admin Dashboard.

The application allows users to use `/report <text>` and `/status` in Discord, handles interactions seamlessly via Ed25519 signature verification, mirrors notifications to a secondary channel, and displays a live log of all commands in a secure dashboard.

## Tech Stack
- **Backend:** Node.js, Express.js, Prisma (PostgreSQL)
- **Frontend:** React, Vite, Tailwind CSS, Framer Motion
- **Database:** Neon PostgreSQL
- **Integrations:** Discord Interactions API, Discord Webhooks

## Directory Structure
- `discord-bot-backend`: Express.js backend (interactions endpoint, REST APIs, Prisma schema)
- `discord-admin-web`: React frontend (Vite, Tailwind, Authentication, live dashboard)

## Local Setup

### 1. Database Setup
1. Create a free PostgreSQL database on Neon or Supabase.
2. Get the connection string (`DATABASE_URL`).

### 2. Discord App Setup
1. Go to the [Discord Developer Portal](https://discord.com/developers/applications) and create a new application.
2. Copy the **Application ID** and **Public Key**.
3. Under the **Bot** tab, generate and copy the **Bot Token**.
4. To test locally and expose your backend to Discord, use **VS Code's Built-in Port Forwarding**:
   - Open the **Ports** tab in the VS Code terminal panel (next to Terminal/Output).
   - Click **Forward a Port** and enter `3001`.
   - Right-click the newly forwarded port and change **Port Visibility** to **Public**.
   - Right-click and copy the **Forwarded Address**.
5. In the Discord Developer Portal, paste your VS Code Forwarded Address + `/api/interactions` into the **Interactions Endpoint URL** field (e.g., `https://<your-vscode-url>/api/interactions`). Discord will send a `PING` to verify it.

### 3. Backend Setup
```bash
cd discord-bot-backend
npm install
```

Copy `.env.example` to `.env` and fill it in:
```env
PORT=3001
DATABASE_URL="postgresql://user:pass@host:5432/db"
DISCORD_APPLICATION_ID="your_app_id"
DISCORD_PUBLIC_KEY="..."
DISCORD_BOT_TOKEN="..."
NOTIFICATION_WEBHOOK_URL="..."
ADMIN_USER="admin"
ADMIN_PASS="password"
JWT_SECRET="secret"
```

Then, initialize the PostgreSQL database, generate the Prisma client, and seed the default Admin user:
```bash
npx prisma generate
npx prisma db push
npm run seed
```

Start the backend server:
```bash
npm run dev
```

### 4. Register Discord Commands
In a separate terminal, register the slash commands (`/report`, `/status`, `/leave`, `/roastme`) with Discord:
```bash
npm run register-commands
```

### 5. Frontend Setup
```bash
cd ../discord-admin-web
npm install
npm run dev
```
Open `http://localhost:5173` in your browser. 
Login using the credentials generated by the seed script:
- **Username:** `admin`
- **Password:** `password`

## Instructions for Testing

> **⚠️ IMPORTANT NOTE FOR GRADERS/TESTERS:**
> Because the backend is deployed on **Render's Free Tier**, the server will automatically go to "sleep" after 15 minutes of inactivity. 
> **When you make your first request (or run your first slash command), it may take 1-2 minutes for the server to wake up.** If your first Discord command fails or the dashboard hangs, please simply wait 60 seconds and try again!

1. **Add Bot to Server:** In the Discord Developer Portal, go to **OAuth2 > URL Generator**. Check `bot` and `applications.commands`. Give the bot `Send Messages` permissions. Copy the generated URL, open it in your browser, and add the bot to your test server.
2. **Run a Command:** Go to your test server and type `/report text:Hello World!`.
3. **Verify Mirroring:** Check the channel configured in `NOTIFICATION_WEBHOOK_URL` to ensure the mirrored notification arrived.
4. **Login to Dashboard:**
   - Go to `http://localhost:5173/login`
   - **Demo Credentials:** Username: `admin`, Password: `password`
5. **View Logs:** Navigate to the Dashboard and watch your command logs update!

## Features
- **/report & /leave**: Prompts a text input or modal, securely logs to database, and triggers an AI triage via Groq.
- **/roastme**: A fun command where the Groq AI playfully roasts the user.
- **AI Triage**: Uses `llama-3.3-70b-versatile` via Groq to analyze discord command inputs and categorize them (e.g., Medical Emergency vs Vacation).
- **Admin Dashboard**: Real-time polling of all discord interactions, JWT-protected routing, and settings toggles to enable/disable commands dynamically.

## Deployment Strategy

This application is built to be deployed for free without a credit card.
1. **Database:** Neon .
2. **Backend (Render):** Deploy the `discord-bot-backend` as a Render Web Service. Ensure you add all `.env` variables in the Render dashboard. Set the Build Command to `npm install && npx prisma db push` and Start Command to `npm start`.
3. **Frontend (Vercel):** Deploy the `discord-admin-web` folder. Set the `VITE_API_URL` to your live backend Render URL.
4. **Update Discord Endpoint:** Once deployed, update your Interactions Endpoint URL in the Discord portal to your live Render backend URL (`https://<your-render-url>.onrender.com/api/interactions`).

## 🛠️ How to Create a Mirror Channel

A mirror channel allows you to forward command logs (like reports and leave requests) to a separate channel in Discord or Slack using a Webhook.

### For Discord
1. Go to your target Discord Server and open the **Channel Settings**.
2. Click on **Integrations** > **Webhooks** > **New Webhook**.
3. Name your webhook and click **Copy Webhook URL**.
4. Paste this URL into your backend `.env` file as `NOTIFICATION_WEBHOOK_URL`.

### For Slack
1. Go to your Slack API workspace and create a new App.
2. Enable **Incoming Webhooks**.
3. Click **Add New Webhook to Workspace** and select the channel.
4. Copy the Webhook URL and paste it into your backend `.env` file as `NOTIFICATION_WEBHOOK_URL`.

## 🧠 AI Integration (Groq)

This bot uses the Groq API (Llama 3) to automatically triage and summarize the contents of `/report` and `/leave` commands.

To enable this:
1. Get a free API key from [Groq Console](https://console.groq.com/).
2. Add `GROQ_API_KEY=your_key_here` to your `.env` file.
3. The AI analysis will automatically appear in the bot's Discord responses and the admin dashboard's **AI Triage** section.
